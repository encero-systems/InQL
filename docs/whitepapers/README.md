# IncQL whitepapers

This directory holds non-normative design papers for larger IncQL product and architecture directions.

Whitepapers are intentionally broader than RFCs. They frame a problem, name a north star, and identify design pressure before a specific normative proposal is ready. When a whitepaper direction becomes implementation work, split the actionable contracts into focused RFCs under `docs/rfcs/`.
